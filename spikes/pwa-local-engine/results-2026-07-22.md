# Local ASC/DBC Engine Results - 2026-07-22

Roadmap milestone: `0.2 - Moteur local ASC DBC`

This implementation adds a fixture-focused browser-local engine:

- TypeScript ASC parser with streaming-compatible scanner.
- DBC subset parser for `BO_` and `SG_` definitions used by current fixtures.
- Little-endian unsigned signal decoding for the fixture DBCs.
- Columnar local store with batched ingest.
- `LocalPwaBackend` facade for status, signals, trace, trace locate, series,
  cursor, frame signals, import job, and purge.
- Minimal static UI at `spikes/pwa-local-engine/index.html`.
- Dependency-free browser build script that strips TypeScript into
  `spikes/pwa-local-engine/browser/*.mjs`.
- Headless Chromium smoke test for the minimal UI with `sample.asc` and
  `sample.dbc`.

## Validation

Commands:

```bash
node spikes/pwa-local-engine/build-browser.mjs
node --test spikes/pwa-local-engine/tests/*.test.ts
CTD_ENGINE_PORT=9881 CTD_ENGINE_DEBUG_PORT=9231 node spikes/pwa-local-engine/browser-smoke.mjs
```

Results:

- Unit/parity tests: 7 passed, 0 failed.
- Browser smoke: passed. The static UI imported `tests/fixtures/sample.asc` and
  `tests/fixtures/sample.dbc` without FastAPI, then rendered:
  - `loaded: true`
  - 6 frames, 2 events, 5 decoded frames, 3 unique ids
  - signals including `EngineData.EngineSpeed`
  - trace rows including `EngineData`
  - first series values `[1024, 2048, 3072, 4096] rpm`

Known limits before a production PWA:

- DBC parsing is intentionally a subset, not full `cantools` parity.
- Motorola byte order, multiplexing, value tables, and broad signed edge cases
  need explicit follow-up tests.
- Durable browser persistence is not final; see
  `spikes/pwa-local-engine/storage-decision-2026-07-22.md`.
- Large-file query performance after semantic indexing still needs benchmark
  coverage beyond the 0.1 scan-only evidence.
