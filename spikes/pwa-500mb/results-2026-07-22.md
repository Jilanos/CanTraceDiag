# PWA 500 MB ASC Spike Results - 2026-07-22

Branch: `feat/pwa-local-first`

Goal: prove whether a browser-local scanner can read large ASC files by chunks
without backend calls and without retaining all frames as JavaScript objects.

## Implementation

- Static spike page: `spikes/pwa-500mb/index.html`
- Browser worker: `spikes/pwa-500mb/asc-worker.mjs`
- Testable scanner core: `spikes/pwa-500mb/scanner-core.mjs`
- Boundary tests: `spikes/pwa-500mb/scanner-core.test.mjs`
- Synthetic fixture generator: `spikes/pwa-500mb/generate-asc-fixture.mjs`
- Node streaming benchmark: `spikes/pwa-500mb/benchmark-node.mjs`
- Headless Chromium benchmark: `spikes/pwa-500mb/browser-benchmark.mjs`

Chunk size used for all measurements: 8 MiB.

## Decision

Go for roadmap milestone 0.2, with constraints:

- Keep parsing in a Web Worker.
- Keep chunk size configurable, defaulting to 8 MiB until broader tests suggest
  otherwise.
- Continue avoiding retained per-frame JavaScript objects during import.
- Treat this as scan feasibility only. DBC parsing, indexed persistence, and
  query performance still need their own milestones.
